## **Moving Directories**

---

```bash
[cp](https://phoenixnap.com/kb/cp-command) -R|--recursive foo bar                               # Copy directory
[mv](https://phoenixnap.com/kb/mv-command-linux) foo bar                                              # Move directory

[rsync](https://phoenixnap.com/kb/rsync-command-linux-examples) -z|--compress -v|--verbose /foo /bar              # Copy directory, overwrites destination
rsync -a|--archive -z|--compress -v|--verbose /foo /bar # Copy directory, without overwriting destination
rsync -avz /foo username@hostname:/bar                  # Copy local directory to remote directory
rsync -avz username@hostname:/foo /bar                  # Copy remote directory to local directory
```

## **Deleting Directories**

---

```bash
rmdir foo                        # Delete empty directory
[rm](https://phoenixnap.com/kb/delete-file-linux) -r|--recursive foo            # Delete directory including contents
rm -r|--recursive -f|--force foo # Delete directory including contents, ignore nonexistent files and never prompt
```

## **Moving Files**

---

```bash
cp foo.txt bar.txt                                # Copy file
mv foo.txt bar.txt                                # Move file

rsync -z|--compress -v|--verbose /foo.txt /bar    # Copy file quickly if not changed
rsync z|--compress -v|--verbose /foo.txt /bar.txt # Copy and rename file quickly if not changed
```

## **Compressing Files**

---

**zip**

```bash
zip foo.zip /bar.txt                # Compress bar.txt into foo.zip
zip foo.zip /bar.txt /baz.txt       # Compress bar.txt and baz.txt into foo.zip
zip foo.zip /{bar,baz}.txt          # Compress bar.txt and baz.txt into foo.zip
zip -r|--recurse-paths foo.zip /bar # Compress directory bar into foo.zip
```

[**gzip**](https://phoenixnap.com/glossary/file-compression)

```bash
[gzip](https://phoenixnap.com/glossary/file-compression) /bar.txt foo.gz           # Compress bar.txt into foo.gz and then delete bar.txt
gzip -k|--keep /bar.txt foo.gz # Compress bar.txt into foo.gz
```

[**tar](https://phoenixnap.com/kb/extract-tar-gz-files-linux-command-line#htoc-using-tar-utility) -c**

```bash
[tar](https://phoenixnap.com/kb/extract-tar-gz-files-linux-command-line#htoc-using-tar-utility) -c|--create -z|--gzip -f|--file=foo.tgz /bar.txt /baz.txt # Compress bar.txt and baz.txt into foo.tgz
tar -c|--create -z|--gzip -f|--file=foo.tgz /{bar,baz}.txt    # Compress bar.txt and baz.txt into foo.tgz
tar -c|--create -z|--gzip -f|--file=foo.tgz /bar              # Compress directory bar into foo.tgz
```

| --- | --- | --- | --- |
| 7 | read, write and execute | rwx | 111 |
| 6 | read and write | rw- | 110 |
| 5 | read and execute | r-x | 101 |

| 2 | write only | -w- | 010 |
| 1 | execute only | - -x | 001 |

**For a directory, execute means you can enter a directory.**

| **User** | **Group** | **Others** | **Description** |
| --- | --- | --- | --- |
| 6 | 4 | 4 | User can read and write, everyone else can read (Default file permissions) |
| 7 | 5 | 5 | User can read, write and execute, everyone else can read and execute (Default directory permissions) |

- u - User
- g - Group
- o - Others
- a - All of the above

```bash
ls -l /foo.sh            # List file permissions
chmod +100 foo.sh        # Add 1 to the user permission
chmod -100 foo.sh        # Subtract 1 from the user permission
chmod u+x foo.sh         # Give the user execute permission
chmod g+x foo.sh         # Give the group execute permission
chmod u-x,g-x foo.sh     # Take away the user and group execute permission
chmod u+x,g+x,o+x foo.sh # Give everybody execute permission
chmod a+x foo.sh         # Give everybody execute permission
chmod +x foo.sh          # Give everybody execute permissio
```

## **Core User Management Commands**

---

### 1. **Adding a User**

- `useradd` or `adduser` (Note: `adduser` is more user-friendly but is often a script wrapper for `useradd` and may not be on all distributions like Red Hat).

**Basic Syntax:**

```bash
sudo useradd [options] <username>

```

**Common Options:**

- `m` or `-create-home`: Create the user's home directory (e.g., `/home/username`).
- `c` or `-comment "Full Name"`: A comment, typically the user's full name.
- `s` or `-shell /bin/bash`: Specify the user's login shell.
- `G` or `-groups`: A comma-separated list of supplementary groups.
- `d` or `-home-dir`: Specify a custom home directory path.
- `u` or `-uid`: Specify a custom User ID (UID).

**Example: Create a user with a home directory and full name:**

```bash
sudo useradd -m -c "John Doe" -s /bin/bash johndoe

```

### 2. **Setting/Changing a User Password**

- `passwd`

**Syntax:**

```bash
sudo passwd <username>  # Set/change another user's password
passwd                  # Change your own password

```

**Example:**

```bash
sudo passwd johndoe

```

### 3. **Modifying a User**

- `usermod`

**Common Options:**

- `c, -d, -m, -s`: Same as `useradd` for changing comment, home dir, etc.
- `G <groups>`: **Replaces** the user's supplementary groups. Use `aG` to *append* instead.
- `aG`: **Append** the user to supplementary groups (crucial to avoid removing existing groups).
- `L` or `-lock`: Lock a user account.
- `U` or `-unlock`: Unlock a user account.
- `l` or `-login`: Change the user's login name.

**Example: Add user 'johndoe' to the 'sudo' and 'www-data' groups:**

```bash
sudo usermod -aG sudo,www-data johndoe

```

### 4. **Deleting a User**

- `userdel`

**Common Options:**

- `r` or `-remove`: Remove the user's home directory and mail spool.

**Example: Delete user and their home directory:**

```bash
sudo userdel -r johndoe
```

- `chage` (Change Age) - Manages user password expiry information.

**Examples:**
